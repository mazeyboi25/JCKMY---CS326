# Support Plan

## Issue Reporting Process
Users can report issues through:
- In-app feedback (future feature)
- Direct message/email to developers

Reported issues should include:
- Description of the problem
- Steps to reproduce
- Screenshot (if possible)

## Response Times
- Critical issues (login, posting, messaging): within 24 hours
- Major issues (UI bugs, incorrect data): within 48 hours
- Minor issues (layout, small glitches): within 3–5 days

## Issue Handling Process
1. Receive and log the issue
2. Reproduce the bug
3. Assign to developer (frontend/backend)
4. Fix and test the issue
5. Deploy fix and confirm resolution

## Supported Features
- Login and verification system
- Viewing and browsing listings
- Posting items for sale
- Messaging between users
